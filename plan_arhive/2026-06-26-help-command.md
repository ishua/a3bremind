# Добавление команды /help

> План реализации команды `/help` для Telegram-бота a3bremindbot.

Без задачи

## Контекст

a3bremindbot — Telegram-бот для напоминаний на Go (github.com/go-telegram-bot-api/v5) с SQLite.
Существующие команды: /start, /settings, /add, /schedule, /list, /skip, /snooze, /pause, /resume, /delete.
Команды регистрируются вручную в `handler.go` через `switch/case` с `strings.HasPrefix`.
Команды живут в отдельных файлах в `internal/bot/` (напр. `pause.go`, `schedule.go`).
Команда `/help` отсутствует; любая неизвестная команда выдаёт "Неизвестная команда".

## Цель

Добавить команду `/help`, которая выводит список всех доступных команд с описанием в Markdown-формате.
Реализовать в виде отдельного файла `internal/bot/help.go` по существующему паттерну.

## Фаза 1: Реализация

- [ ] Создать файл `internal/bot/help.go` с функцией `handleHelp`
  - Функция принимает update, не требует БД (не нужны store/domain)
  - Вызывает `h.sendText` с Markdown-сообщением о всех командах
  - Использовать `tgbotapi.NewMessage` с `ParseMode: tgbotapi.ModeMarkdown`
- [ ] Зарегистрировать команду в `handler.go`:
  - Добавить `case strings.HasPrefix(text, "/help"): h.handleHelp(update)` перед `default`
- [ ] Проверить сборку: `go build ./...`

Текст help (Markdown):

```
*Доступные команды*

/start — начать работу с ботом
/settings timezone Europe/Berlin — установить часовой пояс
/add "Название" daily|once [gap:Nh|Nm] HH:MM ... — создать напоминание
/schedule [tomorrow] — показать расписание
/list — все шаблоны напоминаний
/skip — пропустить активное напоминание
/snooze N — отложить на N минут
/pause — приостановить все напоминания
/resume — возобновить напоминания
/delete ID — удалить напоминание
/help — показать эту справку
```

Почему в этой фазе: задача атомарна — одна новая команда в одном файле с одной регистрацией.

## Решения и договорённости

- **Формат вывода**: Markdown (ParseMode: Markdown). Решение: пользователь выбрал Markdown.
- **Содержание**: все команды с описанием. Решение: пользователь выбрал все команды.
- **/start не меняется**. Решение: пользователь не хочет обновлять /start.
- **Новый файл**: `internal/bot/help.go` по аналогии с `pause.go`. Решение: отдельный файл для каждой команды — конвенция проекта.
- **Не требует БД**: help не обращается к store, это простой текст.
